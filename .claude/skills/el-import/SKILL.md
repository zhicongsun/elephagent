# Import

Use this skill to import existing memories and skills from Claude Code, Cursor, or Codex into the shared `.agent/` directory.

## When to invoke

- The user says "import memories", "import skills", "import from cursor", "import from claude", "import existing config", or similar
- The user wants to import a specific skill by name (e.g. "import the deploy-check skill")
- The user wants to import skills from a specific directory

## Steps

1. Determine what the user wants to import:
   - **Everything** (default): run `elephagent import`
   - **From a specific platform**: run `elephagent import --from claude`, `--from cursor`, or `--from codex`
   - **Specific skills by name**: run `elephagent import skill-name-1 skill-name-2` (searches ~/.cursor/skills/ and ~/.claude/skills/)
   - **From a directory**: run `elephagent import --path /path/to/skills`
2. Read the output and summarize:
   - Which memories were imported and from where
   - Which skills were imported and from where
   - Which items were skipped (already exist)
3. If nothing was imported, explain that all existing content is either already in `.agent/` or was generated by elephagent

## Notes

- Only imports hand-written content — files generated by `elephagent build` are automatically skipped
- Existing files in `.agent/` are never overwritten
- In auto mode, also scans global skill directories (~/.cursor/skills/, ~/.claude/skills/)
- After import, `elephagent build` runs automatically to sync all platforms
